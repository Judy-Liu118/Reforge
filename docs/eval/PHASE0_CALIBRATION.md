## Coverage boundary (not a metric)

- D1″ probes **only the timeout sub-path** of deliberate STOP (`failure_mode == "timeout"`, set in `classifier.py:36-40` when sandbox watchdog kills the process). The **terminal_intentional sub-path** (`is_expected_failure=True AND retryable=False`, set when IntentStage classifies the user request as EXPECTED_ERROR / TRACEBACK_DEMO) is NOT exercised here — constructing a prompt that triggers it would leak intent into the calibration corpus. See `docs/KNOWN_LIMITATIONS.md` L3 for the architectural gap and why a pattern-based detector is deferred.
- BIRD result-direction comparison (governor vs naive solve / recovery rates) is **out of Phase 0 scope** and is intentionally not computed in this report. See Phase 1 (`docs/eval/...` to be written) for the headline ablation.
